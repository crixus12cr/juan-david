comandos basicos de git

git init

El "git init" sirve para inicializar un repositorio para subir el codigo

git remote add "htsttst.com"

El "git remote add "htsttst.com"" se usa cuando se obtiene la url del repositorio para subir a github.

git add .

El "git add ." se usa para subir los cambios de todos los archivos a los cuales se las ha hecho cambios, en este caso a los que aparecen con un -m

git commit

El "git commit" este comando se utiliza para hacer un commit y poner un mensaje -m "cambio", para identifcar el commit o el cambio hecho"

git branch -M main

El "git branch -M main" este comando se hace una primera vez para poner como rama principal a main

git push -u origin main

El "git push -u origin main" este comando se usa una sola vez para subir los cambios al repositorio y ponerlo en la rama principal main

git branch rama-test

El git branch rama-test este comando sirve para crear ramas en este caso para crear la rama "rama-test"

git checkout rama-test

El git checkout rama-test este comando sirve para cambiarse de ramas en este caso se cambia de "main" a "rama-test", y asi sucesivamente se puede cambiar a cualquir rama

git push -u origin rama-test

este comando se usa una primera vez cuando la rama-test no esta en el repositorio se usa para subir cambios y subir la rama, despues de que la rama este arriba ya no mas puedes ejecutar "git push"

git pull

este comando sirve para descargar cambios locales y tambien para descargar cambios entre ramas por ejemplo si estas en main y quieres descargar los cambios de rama-test escribes "git pull origin rama-test"
